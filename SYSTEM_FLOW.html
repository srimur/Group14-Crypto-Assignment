Secure Centralized EV Charging Payment Gateway

BITS F463 Cryptography Term Project - Complete System Flow

Three Entities

Grid Authority Laptop

Central server. Registers franchises and users. Validates transactions (PIN, balance, VMID). Maintains the blockchain. Holds RSA keys.

grid_authority_app.py : port 5000

Charging Kiosk Terminal

Station machine. Encrypts FID with ASCON-128, generates QR codes. Decrypts scanned QR data. Forwards transactions to Grid.

charging_kiosk_app.py : port 5001

EV Owner Device

User's phone. Registers with Grid. Scans QR codes from kiosk. Sends VMID + PIN + amount to pay for charging.

user_device_app.py : port 5002
Registration Flow
1
GRID AUTHORITY
Franchise owner registers: provides name, zone code, password, initial balance
FID = SHA3(name + timestamp + password)[:16 hex chars]
Password stored as SHA3(password)
|
2
USER DEVICE -> GRID
EV owner registers: provides name, zone code, password, 4-digit PIN, mobile number, balance
UID = SHA3(name + timestamp + password)[:16]
VMID = SHA3(UID + mobile)[:16]
PIN stored as SHA3(pin)
Complete Transaction Flow
1
KIOSK
Kiosk fetches franchise list from Grid, operator selects one
GET http://127.0.0.1:5000/api/franchises
|
2
KIOSK (ASCON-128)
Encrypts FID with ASCON-128 using random nonce. Generates QR code image containing encrypted data.
nonce = os.urandom(16)
ciphertext, tag = ascon_encrypt(key, nonce, fid, associated_data)
QR = session_id | ciphertext_hex | tag_hex | ad_hex
|
3
USER DEVICE
User selects QR session (or scans QR from kiosk screen), picks their account, enters PIN and amount
QR data obtained from dropdown or camera scan
|
4
USER DEVICE -> KIOSK
Device sends QR data + VMID + PIN + amount to the Kiosk
POST http://127.0.0.1:5001/api/process_session
body: {qr_data, vmid, pin, amount}
|
5
KIOSK (ASCON-128 DECRYPT)
Checks if session is expired (5 min) or already used. Decrypts QR data using ASCON with stored nonce to recover original FID.
fid = ascon_decrypt(key, nonce, ciphertext, tag, ad)
If tag mismatch -> reject (tampered QR)
|
6
KIOSK -> GRID
Forwards decrypted FID + VMID + PIN + amount to the Grid Authority
POST http://127.0.0.1:5000/api/process_transaction
body: {fid, vmid, pin, amount}
|
7
GRID - VALIDATION
Grid validates everything in order:
1. Does FID exist and is franchise active?
2. Does any user have this VMID? Is user active?
3. Does SHA3(entered_pin) == stored pin_hash?
4. Is user balance >= amount?
|
8
GRID - FUND TRANSFER
Deducts amount from user balance, credits franchise balance
user["balance"] -= amount
franchise["balance"] += amount
|
9
BLOCKCHAIN
New block created and appended to chain
txn_id = SHA3(uid + fid + timestamp + amount)
block_hash = SHA3(json of all block fields)
previous_hash = last_block.hash
dispute_flag = False
|
10
HARDWARE FAILURE CHECK (10% random)
If hardware fails: reverse the transfer, add a dispute block with negative amount
user["balance"] += amount (refund)
franchise["balance"] -= amount
new block with amount=-X, dispute=True
|
11
KIOSK
Receives Grid response. Marks session as used. Relays result to User Device.
qr_sessions[sid]["used"] = True
QR image deleted on next cleanup cycle
|
12
USER DEVICE
User sees result: approved (new balance shown), rejected (error message), or refund (hardware failure)
Where Each Algorithm Is Used

ASCON-128 (Lightweight Crypto)

Encrypt: Kiosk encrypts FID -> VFID for QR code
Decrypt: Kiosk decrypts QR data -> recovers FID
Auth tag: Detects if QR data was tampered
File: crypto_utils/ascon.py
Key: 128-bit (config.py)
Nonce: 128-bit random per session
Tag: 128-bit authentication

SHA-3 (Keccak-256)

FID/UID: SHA3(name+time+pass)[:16]
VMID: SHA3(UID+mobile)[:16]
PIN hash: SHA3(pin) stored at registration
Password hash: SHA3(password) stored
Transaction ID: SHA3(uid+fid+time+amount)
Block hash: SHA3(json of block data)
File: crypto_utils/sha3_utils.py
Uses hashlib.sha3_256

RSA (Public Key Crypto)

Key gen: Grid generates (e,n) and (d,n) at startup
Encrypt: User encrypts PIN and VMID with public key
Decrypt: Grid decrypts with private key
File: crypto_utils/rsa_utils.py
Key size: 20 bits (for Shor's demo)
e = 65537, d = mod_inverse(e, phi)

Shor's Algorithm (Quantum Attack)

Input: RSA modulus n from Grid's public key
Does: Factors n into p,q using period-finding
Output: Recovers private key d, proves RSA is broken
File: crypto_utils/shor_simulation.py
Classical simulation (brute-force order finding)
Quantum step would use QFT

Blockchain

Record: Each successful txn = new block
Link: Each block stores previous block's hash
Validate: Recompute hashes, check linkage
Dispute: Hardware failure = refund block (negative amount, dispute=True)
File: blockchain/ledger.py
Genesis block created at Grid startup
Centralized (Grid is sole maintainer)
Edge Cases Handled
X
Wrong PIN
SHA3 hash mismatch. Rejected but session stays valid for retry.
X
Low balance
Grid checks balance before deducting. Shows available vs required.
X
Invalid VMID
No user found with that VMID. Transaction rejected.
X
Tampered QR
ASCON tag verification fails. Decryption rejected.
X
Expired QR (>5 min)
Kiosk checks timestamp. Rejects with elapsed time.
X
Reused QR
Kiosk checks used flag. Single-use only.
X
Hardware failure
10% chance. Funds reversed. Dispute block added to chain.
X
Inactive account
active=False on franchise or user. Transaction blocked.
X
Duplicate registration
Same name+zone rejected for both franchises and users.
X
Blockchain tampered
validate_chain() recomputes hashes and checks linkage.
X
Grid/Kiosk offline
ConnectionError caught. User sees error message, no crash.
X
Bad zone code / PIN format
Validated during registration. Rejected with specific error.
